# 核心小隊（4-5 人）

## 原則

- 上、中、下 + 廣告 = 4-5 人
- 前端跟後端深度綁定
- 每個角色有主要 + 代理人
- 第一案由統籌親跑全程

## 角色定義

### 統籌 + PM
- 職責：場景定義、方向把關、盤點痛點、兩端對焦
- 要求：懂前端（客戶痛點）又懂後端（工程 spec）
- 交付物：痛點清單 + 可行性評估

### BD（業務開發）
- 職責：找到客戶、初步接觸、客戶分級
- 交付物：客戶基本資料 + 初步痛點描述
- 代理機制：主要 BD 離開時，副主席可接手

### 講師 / 顧問
- 職責：針對痛點教學 + 深度規劃解決方案
- 交付物：客戶理解報告 + 解決方案書
- 可一人兼任或拆開

### 工程
- 職責：小型快贏 → 大型建置
- 交付物：上線系統 + 操作教學 + 維運交接

### 廣告品牌
- 職責：全程支援行銷（短影片、Podcast、帶貨）
- 交付物：行銷素材 + 案例展示內容
