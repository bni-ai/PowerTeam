# Power Team 組建方法

## 方法一：三層引薦重疊（由主席/資訊長角度）

### 步驟

1. 蒐集每位成員的三層引薦（一般 / 理想 / 夢幻）
2. 盤點所有人的引薦，找出客群重疊
3. 將客群粗分為 8-15 種樣貌
4. 每種樣貌標出：第一層、第二層、第三層分別是誰
5. 客群重疊的人 = 潛在 PT 夥伴 → 安排一對一確認

### 目的

讓成員知道「原來我跟誰是同樣的客群，他可能是我的上游或下游」。

## 方法二：自我探索（由個人角度）

每個成員問自己：

1. 我做什麼最輕鬆、最省力？（= 我在產業鏈的哪個位置）
2. 我的夢幻客戶長什麼樣？（= 我想服務的客群）
3. 做完我這段，客戶接下來需要什麼？（= 我下游是誰）
4. 客戶在找到我之前，需要先經過誰？（= 我上游是誰）

## 切片化原則

### 核心邏輯

不把所有工作塞進一個 PT。切小、快交付、快結案、快收錢。

### 錯誤做法

一個 PT 包：資料庫整理 + FB 發文 + AI 影像 + 全部
→ 工時長、前後端溝通成本高、交付拖很長

### 正確做法

5-6 個小 PT，每個只接一種明確的事：
- PT-顧問組：診斷完了 → 收一次錢 → 轉介
- PT-講師組：教完了 → 收一次錢 → 轉介
- PT-工程組：系統做完了 → 收一次錢

### 轉介邏輯

客戶滿意 → 介紹給下一個 PT 繼續做。
不是一個 team 從頭做到尾。

## 華AI 的組 PT 困境與解法

### 困境

成員能力重疊 → 分工不明確 → 按產業別分行不通

### 解法

按「服務類型」分，不按「產業別」分：
- 顧問類（診斷、規劃、諮詢）
- 講師類（教學、內訓、課程）
- 工程類（系統開發、AI 導入、自動化）
- 行銷類（廣告投放、品牌、內容）

每人選自己最輕鬆的那個類型站位。不同案子可以換位置。
