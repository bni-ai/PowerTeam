# 盲點分析

## 6 框架分析結果

### BWPA
- Best：第一個美業案例成功 → 複製到其他產業
- Worst：補助沒過 + 快贏做完客戶不續約 + 成員散掉
- Probable：補助慢、快贏有效、部分成員積極部分觀望
- Action：✅ 已定義零補助備案（課程 + 顧問月費 30K）

### 黑天鵝
- 政府砍補助 → ✅ 零補助也能活
- 核心 BD 離開 → ✅ 馬驊代理人機制
- 免費 AI 工具出現 → ⚠️ 快贏要包含「數據 + 洞察」不只是功能

### 灰犀牛
- 分工不均 → ✅ 客戶分級 ABC，C 不接
- 交接斷裂 → ⚠️ 需定義交接模板
- 客戶期望管理 → ✅ 課程與系統拆開
- 利潤分配 → 暫緩，先確認統籌份額再跟成員談

### BCG 矩陣
- 金牛：補助培訓課程（穩定現金流）
- 明星：AI Agent 產品化 / 舊客提醒 SaaS（長期方向）
- 問號：大型系統建置
- 瘦狗：純硬體轉賣（不做）

### 喬哈里視窗
- 隱藏區太大：客戶不知道有完整服務鏈、不知道有補助
- 對策：需做「一頁說明書」給 BD 用

### 芒格思維模型
- 反轉思考：第一案統籌親跑全程
- 激勵偏差：引薦費應跟客戶完成度掛鉤
- 能力圈：4-5 人核心小隊
- 複利效應：第一案做到可展示、可量化
- 歸因謬誤：逐案歸因（補助 vs 能力）
