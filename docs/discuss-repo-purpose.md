> 日期：2026-06-09

## 主要目標

**協助 BNI 協會的成員，打造屬於他們自己的 Power Team。**

## Repo 結構

```
PowerTeam/
├── docs/              ← 給人看的（PT 組建手冊、操作指南）
├── knowledge/         ← 給 AI 看的（結構化 MD，frontmatter + 規則）
└── skills/            ← AI 工具（讀 knowledge/ 來執行任務）
```

- knowledge 是 SSOT，docs 手冊從 knowledge 產出

## 華AI PT vs 傳統 PT

- 傳統 PT：鎖定一個受眾，成員按產業鏈上下游分工（線性接力）
- 華AI PT：成員專業是「AI/系統/數位轉型」，橫切所有產業
- **SOP 架構不變，對接內容隨產業調整**

## 服務交付 SOP — 完整流程

### 頭（入口）

| 項目 | 內容 |
|------|------|
| 人群 | 紋繡/美甲/美睫工作室老闆（1-3 人小店） |
| 場景 | 老闆身兼技術師、行銷、會計、客服 |
| 痛點 | 時間全被瑣事吃掉（預約、記帳、call 客） |
| 交付物 | 痛點清單 + 可行性評估 |

痛點拆解：

```
美業老闆的一天
├── 技術（本業，不可取代）— 種眉、修眉、美甲、紋唇
├── 行銷（耗時，可半自動化）— 拍片、社群、call 客
└── 行政（耗時，可全自動化）— 預約排程、記帳、客戶資料整理
```

### 中間（雙軌並行）

**流程順序**：先診再教（路線 B）— PM 先盤點痛點，講師再針對性教學。

**課程與系統拆開**：教課歸教課，系統歸系統，兩個獨立報價。

**補助軌**：Lynn 啟動政府 AI 培訓補助申請（背景跑，等審核）
- 講師費：2,000/hr × 30hr = 60,000/案
- 引薦費：20,000/案
- AI 工具費：GPT/Claude/token 費用可報

**零補助備案**：課程導入為主，顧問月費制（約 30,000/月），老闆自費。補助是加速器不是引擎。

**執行軌**：

```
① BD 找到客戶 → 客戶資料 + 初步痛點
     ↓
② PM 盤點 → 痛點清單 + 現況流程圖
     ↓
③ 講師針對痛點教學 → 客戶理解哪些可自動化
     ↓
④ 顧問深度規劃 → 解決方案書（做什麼、用什麼、優先順序）
     ↓
⑤ 工程小型快贏 → 舊客自動提醒系統（上線 + 教學）
     ↓                客戶看到：省時間、客戶回流、有營收
  補助下來 →
     ↓
⑥ 大型建置 → 預約Bot + 自動記帳 + 完整CRM
     ↓
  驗收報告 + 維運交接
```

### 尾（出口）

| 項目 | 內容 |
|------|------|
| 狀態 | 痛點已被系統解決，穩定運行 |
| 交付物 | 上線系統 + 驗收報告 + 維運交接 |
| 客戶感受 | 美業老闆只做技術，其他讓系統跑 |

## 核心小隊（4-5 人，不是 35 人）

**原則**：前端跟後端深度綁定，不深度擴張。上、中、下 + 廣告 = 4-5 人。

**第一案必須由老魚親自跑完全程**，跑通後才能複製。

**備援機制**：主要負責人 + 代理人概念。BD 斷鏈 → 馬驊（副主席）可接手。

## 客戶分級（ABC + D）

| 等級 | 特徵 | 策略 |
|------|------|------|
| **D（黃金）** | 有錢、有時間、知道要什麼、想擴張 | 全力投入，優先服務 |
| **A** | 有錢、知道要什麼、沒時間 | 主力客群，快速交付 |
| **B** | 知道要什麼、沒錢或不想付、沒時間 | 課程先行，驗證意願再投入 |
| **C** | 不知道要什麼、不想付、沒時間 | 不接，或只提供免費內容篩選 |

**判斷標準**：案子進來先分級，C 不接。避免工程端瓶頸（接太多 B/C 客戶消耗資源）。

## 盲點分析（6 框架）

### 1. BWPA

| 情境 | 分析 |
|------|------|
| Best | 第一個美業案例成功 → 複製到其他產業 |
| Worst | 補助沒過 + 快贏做完客戶不續約 + 成員散掉 |
| Probable | 補助慢、快贏有效、部分成員積極部分觀望 |
| Action | ✅ 已定義零補助備案（顧問月費 30K） |

### 2. 黑天鵝

| 事件 | 對策 |
|------|------|
| 政府砍補助 | ✅ 零補助也能活（課程 + 顧問費） |
| 核心 BD 離開 | ✅ 馬驊代理人機制 |
| 免費 AI 工具出現 | ⚠️ 快贏要包含「數據 + 洞察」不只是功能 |

### 3. 灰犀牛

| 風險 | 對策 |
|------|------|
| 分工不均 | ✅ 客戶分級 ABC，C 不接 |
| 交接斷裂 | ⚠️ 需定義交接模板（引擎 4 缺口） |
| 客戶期望管理 | ✅ 課程歸課程、系統歸系統，拆開報價 |
| 利潤分配 | 暫緩，先確認老魚自己要拿多少，再跟成員討論 |

### 4. BCG 矩陣

| 象限 | 產品 | 策略 |
|------|------|------|
| 金牛 | 補助培訓課程 | 穩定現金流，但不能只停在這裡 |
| 明星 | AI Agent 產品化 / 舊客提醒 SaaS | **長期要往這裡走** |
| 問號 | 大型系統建置 | 看第一案結果決定 |
| 瘦狗 | 純硬體轉賣 | 不做 |

### 5. 喬哈里視窗

- **隱藏區太大**：客戶不知道我們有完整服務鏈、不知道有補助
- **對策**：⚠️ 需做「一頁說明書」給 BD 用
- **內容方向**：痛點共鳴（你是不是...）→ 我們怎麼解 → 一個成功數字

### 6. 查理芒格思維模型

| 模型 | 結論 |
|------|------|
| 反轉思考 | ✅ 第一案老魚親跑全程 |
| 激勵偏差 | ⚠️ 引薦費應跟客戶完成度掛鉤（待定義） |
| 能力圈 | ✅ 4-5 人核心小隊，不硬塞 35 人 |
| 複利效應 | ✅ 第一案做到可展示、可量化（省幾小時、回流幾個客戶） |
| 歸因謬誤 | ✅ 案例完成後逐案歸因：補助貢獻 vs 能力貢獻 |
