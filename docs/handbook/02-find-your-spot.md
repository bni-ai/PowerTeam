# 02 — 找到你的位置

## 核心原則

沒有人幫你指定位置。你自己選。
標準只有一個：**做起來最輕鬆、最省力的那件事**。

做完你最輕鬆的那段，剩下的交給 PT 夥伴。
你專心做你擅長的，其他人做他們擅長的。

## 自我探索四問

拿一張紙，回答這四個問題：

### Q1：我做什麼最輕鬆？

想想你過去的工作經驗，哪件事你做起來不覺得累、甚至覺得有趣？

- 跟人聊天、挖需求 → 你可能適合「顧問」
- 站在台上講課 → 你可能適合「講師」
- 坐下來寫程式、搞系統 → 你可能適合「工程」
- 做素材、拍影片、投廣告 → 你可能適合「行銷」
- 到處認識人、牽線搭橋 → 你可能適合「BD（業務開發）」

### Q2：我的理想客戶長什麼樣？

具體描述你最想服務的人：

- 他是什麼產業？
- 他公司多大？（一人、5 人、50 人、500 人？）
- 他願意付多少錢？
- 他的痛點是什麼？

例：「1-3 人的美業工作室老闆，每月願意付 1-2 萬顧問費，痛點是時間都花在雜事上。」

### Q3：做完我這段，客戶接下來需要什麼？

你做完你的事，客戶下一步要什麼？那就是你的**下游**。

- 我教完課 → 客戶想導入系統 → 下游 = 工程
- 我做完診斷 → 客戶需要有人教員工 → 下游 = 講師
- 我投完廣告 → 客戶進來了，需要有人接待 → 下游 = 顧問/BD

### Q4：客戶在找到我之前，需要先經過誰？

誰把客戶帶到你面前？那就是你的**上游**。

- 客戶先聽了分享會才來找我 → 上游 = 講師
- 客戶是被朋友介紹來的 → 上游 = BD
- 客戶先做了諮詢才決定要做系統 → 上游 = 顧問

## 畫出你的位置圖

把四個答案填進去：

```
我的上游：___________（誰把客戶帶給我）
        │
        ▼
我做的事：___________（我最輕鬆的那件事）
        │
        ▼
我的下游：___________（客戶做完我這段之後需要誰）
```

## 一人多位置

你不一定只站一個位置。

不同案子、不同客群，你可能換位置：
- A 案裡你是顧問（營運 PT）
- B 案裡你是講師（學習 PT）
- 幫某個夥伴拉客戶時，你是他的營銷 PT

這很正常。選你最輕鬆的當「主力位置」，其他當「客串」。

## 不想做的事，就不做

如果某件事做起來很痛苦，那就不是你的位置。
即使你「會做」，不代表你「該做」。

例：你會架網站，但每次接網站案都覺得很累 → 那就不站工程位，讓別人做。

Power Team 的意義就是：你只做你最輕鬆的，其他交出去。
