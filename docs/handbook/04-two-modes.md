# 04 — 兩種運作模式

Power Team 有兩種運作方式。不是二選一，是前後銜接。

## 模式 A：產業鏈

客戶有一個明確需求，沿著鏈走，每段由不同人做。

```
客戶需求進來
    │
    ▼
顧問諮詢（診斷痛點、出報告）
    │ 交付：痛點清單 + 可行性評估
    ▼
講師教學（教客戶理解 AI 能幫什麼）
    │ 交付：客戶知道哪些可以自動化
    ▼
員工培訓（教客戶的員工實際操作）
    │ 交付：員工會用工具
    ▼
系統導入（幫客戶建置系統）
    │ 交付：上線的系統 + 操作手冊
    ▼
客製產品（進階需求：客製化開發）
    │ 交付：完整解決方案
    ▼
  結案
```

**特徵：**
- 一個客戶，多人接力
- 每人只做自己擅長的段落
- 做完一段 → 交付 → 收錢 → 轉介下一段
- 不是每案都走完整條鏈，按需求切

**適合場景：**
- 客戶已經知道自己要什麼
- 需求明確、可以拆成階段
- 例：保養品公司要做 AI 行銷 → 顧問先診斷 → 講師教操作 → 工程做系統

## 模式 B：講師聯盟

全員都是講師，一起曝光獲客，再按需求拆成產業鏈。

```
全員各有講課主題
    │
    ├─ 線上課程（各自錄、平台上架）
    ├─ 免費分享會（輪流講、聯合辦）
    ├─ 實體交流會（定期聚、互相學）
    └─ 企業內訓（聯合接案、按主題分工）
           │
           ▼
     企業聽完有具體需求
           │
           ▼
     切換到「模式 A 產業鏈」做對接
```

**特徵：**
- 前端一起曝光、獲客
- 每人講自己的專業主題
- 客戶聽完課產生需求 → 再拆到產業鏈去做
- 低成本啟動（不需要先有系統或產品）

**適合場景：**
- 剛開始，還沒有明確客戶
- 想要先建立知名度和信任感
- 例：5 個講師辦「中小企業 AI 應用」系列分享會 → 聽眾裡有老闆想導入 → 轉產業鏈

## 兩種模式的關係

```
模式 B（講師聯盟）= 入口漏斗
        │
        │ 客戶聽完課，產生具體需求
        ▼
模式 A（產業鏈）= 交付引擎
```

- 講師聯盟負責「讓客戶認識我們」
- 產業鏈負責「把事情做完」
- 先用 B 獲客，再用 A 交付

## 最小成本啟動

不需要先建系統、先有產品。最小成本的啟動方式：

| 階段 | 做什麼 | 成本 |
|------|--------|------|
| 第一步 | 開課 / 辦分享會 | 幾乎零成本（講就行） |
| 第二步 | 接顧問諮詢 | 零成本（用嘴巴和腦袋） |
| 第三步 | 才做系統開發 | 有成本（需要工時） |

課程和顧問是最不花腦力、最快能收錢的起點。
系統開發等到客戶真的要了，再啟動。
