# 06 — 案例

## 案例一：美業老闆的 AI 導入

### 背景

紋繡工作室老闆，1 人店。
每天：做眉毛 → 拍短影片 → 回 LINE 訊息 → 手動記帳 → Call 舊客。
痛點：時間全被瑣事吃掉，沒時間接更多客人。

### PT 怎麼運作

```
BD 在分享會上認識這位老闆
  │
  ▼ 交付：客戶基本資料 + 「她很忙、想省時間」
  │
顧問（老魚）一對一診斷
  │ 發現：最痛的是「舊客流失」— 做完一次就不回來
  │
  ▼ 交付：痛點清單 + 「先做舊客自動提醒，ROI 最高」
  │
講師教她：什麼是自動提醒、怎麼運作、她要準備什麼
  │
  ▼ 交付：老闆理解了、同意做
  │
工程（Luka）建置舊客自動提醒系統
  │ 回訪提醒 + 生日提醒
  │
  ▼ 交付：上線系統 + 操作教學
  │
結案。老闆只做眉毛，提醒系統自己跑。
```

### 每人花的時間

| 角色 | 花的時間 | 收費 |
|------|---------|------|
| BD | 1 次分享會 + 1 次一對一 | 引薦費 |
| 顧問 | 1-2 次診斷（共 2-3 小時） | 顧問費 |
| 講師 | 1 次教學（2 小時） | 講師費 |
| 工程 | 建置 + 上線（約 1-2 週） | 系統費 |

### 關鍵

- 每人只做自己最擅長的那段
- 做完就交棒，不拖泥帶水
- 快贏（舊客提醒）先上 → 客戶有感 → 後續需求自然來

## 案例二：講師聯盟接企業內訓

### 背景

一間中型公司（50 人）想做 AI 內訓。
需求模糊：「老闆聽說 AI 很重要，想讓員工學。」

### PT 怎麼運作

```
講師聯盟裡有 4 位講師，各有主題：
  ├─ A：AI 工具應用（ChatGPT、Midjourney）
  ├─ B：AI 行銷（社群、廣告素材）
  ├─ C：AI 資料整理（知識庫、自動化流程）
  └─ D：AI 簡報與溝通

聯合提案 → 企業買 4 堂內訓課程
每人各講 1 堂 → 各收各的講師費

  內訓結束後
      │
      ▼
  企業發現：「我們的客服資料很亂，想整理」
      │
      ▼
  講師 C 轉介給顧問 → 顧問診斷 → 工程做系統
  （切換到產業鏈模式）
```

### 關鍵

- 講師聯盟是入口：4 個人一起比 1 個人容易接到案子
- 每人只講自己的主題，不硬撐不擅長的
- 內訓完 → 自然產生後續需求 → 轉到產業鏈

## 案例三：一人多 PT

### 老魚的三個 PT 位置

```
PT-美業顧問組
  位置：營運（中間）
  做的事：顧問診斷 + 月費諮詢
  上游：BD 找到美業老闆
  下游：講師教操作、工程做系統

PT-講師聯盟
  位置：學習（下游）
  做的事：教中小企業主用 AI
  上游：分享會獲客
  下游：有需求 → 轉到產業鏈

PT-佑禎旅宿鏈
  位置：營銷（上游）
  做的事：教 AI 給設計師/房仲 → 引薦給佑禎
  上游：老魚自己的人脈
  下游：佑禎接手做旅宿轉型
```

### 關鍵

- 同一個人，三個 PT，三個不同位置
- 每個 PT 裡做的事都是「最輕鬆的那件」
- 不會互相衝突，因為服務的客群不同
