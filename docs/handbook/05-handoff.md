# 05 — 交接流程

## 為什麼需要交接？

PT 是接力賽，不是一個人跑全程。
你做完你的段落，要清楚地把棒子交給下一個人。

交接不清楚 → 客戶體驗斷裂 → 客戶流失 → 大家白做。

## 交接三要素

每次交接，確認三件事：

```
1. 我做完了什麼？（交付物）
2. 客戶現在的狀態是什麼？（客戶摘要）
3. 下一個人要做什麼？（接手指引）
```

## 各角色的交付物

### BD → 顧問/PM

| 項目 | 內容 |
|------|------|
| 客戶基本資料 | 公司名、聯絡人、產業、規模 |
| 初步痛點 | 客戶自己說的問題是什麼 |
| 客戶分級建議 | 初判 A/B/C/D（參考客戶分級標準） |
| 溫度 | 客戶急不急？預算概念？決策者是誰？ |

### 顧問/PM → 講師

| 項目 | 內容 |
|------|------|
| 痛點清單 | 每個痛點標註：現況做法、耗時、影響 |
| 可行性評估 | 哪些可以自動化、優先順序建議 |
| 客戶理解程度 | 客戶對 AI 理解到什麼程度？要從哪裡教起？ |

### 講師 → 工程

| 項目 | 內容 |
|------|------|
| 客戶已學會的 | 上完課後客戶會操作什麼 |
| 客戶要做的系統 | 具體需求（功能清單 + 優先順序） |
| 預算範圍 | 客戶可接受的預算區間 |
| 時程期望 | 客戶期望什麼時候上線 |

### 工程 → 客戶（結案）

| 項目 | 內容 |
|------|------|
| 上線系統 | 可運行的系統 |
| 操作教學 | 教客戶/員工怎麼用 |
| 維運說明 | 誰負責後續維護、怎麼報修 |

## 轉介流程

做完你的段落，客戶想要下一階段的服務：

```
1. 跟客戶說明：「接下來這個部分，我的夥伴 ___ 更擅長。」
2. 建立三方群組（你 + 客戶 + 下一位夥伴）
3. 在群組裡把交付物和客戶摘要交代清楚
4. 下一位夥伴接手後，你退到背景（但客戶知道你還在）
```

**重點**：不要把客戶丟出去就不管了。
轉介是「我把你交給更適合的人」，不是「我不想做了」。

## 轉介不等於失去客戶

你轉介出去的客戶，仍然認識你。
下次他有你擅長的需求，他會回來找你。

而且：你幫夥伴轉介 → 夥伴也幫你轉介 → 引薦循環。
